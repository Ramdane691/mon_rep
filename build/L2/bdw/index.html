Algèbre relationnel

L'algèbre relationnelle couvre les concepts fondamentaux des bases de données relationnelles, y compris les opérations de sélection, de projection, de jointure et d'union.

Ces opérations permettent de manipuler et d'interroger les données stockées dans des tables relationnelles.

Voici un aperçu des principales opérations de l'algèbre relationnelle :

  • Sélection (σ) : permet de filtrer les lignes d'une table en fonction d'une condition.
  • Projection (π) : permet de sélectionner certaines colonnes d'une table.
  • Jointure (⨝) : permet de combiner des lignes de deux tables en fonction d'une condition.
  • Union (∪) : permet de combiner les lignes de deux tables, en éliminant les doublons.
  • Différence (-) : permet de soustraire les lignes d'une table à celles d'une autre.
  • Produit cartésien (×) : permet de combiner toutes les lignes de deux tables.
  • Renommage (ρ) : permet de renommer une table ou une colonne.
  • Intersection (∩) : permet de trouver les lignes communes entre deux tables.

Les étudiants apprendront à écrire des requêtes en utilisant l'algèbre relationnelle pour extraire des informations pertinentes à partir de bases de données.

Exemple d'expression algébrique :

σ (age > 18) (étudiants)

Cette expression sélectionne toutes les lignes de la table "étudiants" où l'âge est supérieur à 18.

SQL

Le SQL (Structured Query Language) est le langage standard utilisé pour interagir avec les bases de données relationnelles.

Il permet de créer, lire, mettre à jour et supprimer des données dans une base de données.

Voici un aperçu des principales commandes SQL :

  • SELECT : permet de sélectionner des données à partir d'une ou plusieurs tables.
  • INSERT : permet d'ajouter de nouvelles lignes dans une table.
  • UPDATE : permet de modifier des lignes existantes dans une table.
  • DELETE : permet de supprimer des lignes d'une table.
  • CREATE TABLE : permet de créer une nouvelle table dans la base de données.
  • ALTER TABLE : permet de modifier la structure d'une table existante.
  • DROP TABLE : permet de supprimer une table de la base de données.
  • JOIN : permet de combiner des lignes de deux tables en fonction d'une condition.
  • WHERE : permet de filtrer les résultats d'une requête en fonction d'une condition.
  • ORDER BY : permet de trier les résultats d'une requête.
  • GROUP BY : permet de regrouper les résultats d'une requête en fonction d'une ou plusieurs colonnes.
  • HAVING : permet de filtrer les résultats d'une requête après le regroupement.
  • UNION : permet de combiner les résultats de deux requêtes.
  • INTERSECT : permet de trouver les lignes communes entre deux requêtes.
  • FROM : permet de spécifier la table à partir de laquelle sélectionner des données.
  • AS : permet de renommer une table ou une colonne dans une requête.
  • LIMIT : permet de limiter le nombre de résultats retournés par une requête.

Voici des exemples d'utilisation : 

SELECT nom, age FROM étudiants WHERE age > 18;

Cette requête sélectionne les colonnes "nom" et "age" de la table "étudiants" où l'âge est supérieur à 18.

INSERT INTO étudiants (nom, age) VALUES ('Alice', 20);

Cette requête insère une nouvelle ligne dans la table "étudiants" avec le nom "Alice" et l'âge 20.

UPDATE étudiants SET age = 21 WHERE nom = 'Alice';

Cette requête met à jour l'âge de l'étudiant nommé "Alice" dans la table "étudiants".

DELETE FROM étudiants WHERE nom = 'Alice';

Cette requête supprime l'étudiant nommé "Alice" de la table "étudiants".

Modélisation

La modélisation des données est le processus de création d'un modèle conceptuel qui représente les données et leurs relations dans une base de données.

Les étudiants apprendront à créer des diagrammes entité-association (DEA) pour représenter les entités, les attributs et les relations entre elles.

Voici un aperçu des concepts clés de la modélisation :

  • Entité : un objet ou un concept du monde réel que l'on souhaite représenter dans la base de données.
  • Attribut : une caractéristique ou une propriété d'une entité.
  • Relation : une association entre deux ou plusieurs entités.
  • Cardinalité : indique le nombre minimum et maximum d'occurrences d'une entité dans une relation.
  • Clé primaire : un attribut ou un ensemble d'attributs qui identifie de manière unique chaque occurrence d'une entité.
  • Clé étrangère : un attribut qui établit une relation entre deux tables en faisant référence à la clé primaire d'une autre table.

Les étudiants apprendront à créer des diagrammes entité-association (DEA) pour représenter les entités, les attributs et les relations entre elles.

Voici un exemple de diagramme entité-association :

Étudiant (id, nom, age)
Cours (id, nom, crédits)
Inscription (id_étudiant, id_cours)

Dans cet exemple, nous avons trois entités : "Étudiant", "Cours" et "Inscription".

La relation "Inscription" relie les étudiants aux cours auxquels ils sont inscrits.
